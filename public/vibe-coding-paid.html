Track 02 — 유료 사용자

AI가 직접
파일을 수정

Cursor, Windsurf, Claude Pro처럼 더 강력한 AI 환경이라면, 단순한 코드 생성을 넘어 프로젝트 전체를 AI와 함께 설계하고 완성할 수 있습니다. 대화하듯 개발하는 진짜 바이브코딩입니다.

Cursor Windsurf Claude Pro Vite + React GitHub Vercel
무료 vs 유료

뭐가 달라지나요?

유료 트랙의 핵심 차이는 단순히 "AI 성능이 좋아지는 것"이 아닙니다. AI가 내 프로젝트 파일을 직접 보고, 읽고, 수정할 수 있다는 것입니다. 복붙 없이 대화만으로 파일이 바뀝니다.

항목 무료 트랙 유료 트랙
AI의 파일 접근 없음 (코드를 복붙해야 함) 직접 읽고 수정 가능
맥락 유지 매번 코드를 다시 붙여줘야 함 프로젝트 전체를 AI가 기억
여러 파일 작업 하나씩만 가능 여러 파일 동시 수정 가능
리팩토링 번거로움 구조 전체 한 번에 개선 요청 가능
디버깅 속도 오류 복사 → 붙여넣기 반복 AI가 코드 읽고 바로 수정

주의: 유료 환경이 편한 만큼, "AI가 뭘 바꿨는지" 모르고 넘어가기 쉽습니다. AI가 변경한 내용을 반드시 직접 확인하고 이해하는 습관이 더 중요해집니다.

Step 1 — 도구 선택 & 설치

어떤 도구를 쓸까요?

유료 트랙에서 쓸 수 있는 도구는 크게 세 가지입니다. 상황에 맞게 선택하세요.

🖱️

Cursor

VS Code 기반의 AI 편집기. 가장 많이 씁니다. cursor.com에서 무료로 설치하고, 무료 플랜도 어느 정도 쓸 수 있습니다.

🌊

Windsurf

Codeium이 만든 AI 편집기. Cursor와 비슷합니다. codeium.com/windsurf에서 설치합니다.

🤖

Claude Pro

구독하면 Projects 기능으로 파일을 업로드하고 맥락을 유지할 수 있습니다. 월 $20 구독.

⚡

GitHub Copilot

VS Code에 플러그인으로 설치. 학생이면 무료. education.github.com에서 신청 가능합니다.

추천: 처음 시작한다면 Cursor 무료 플랜을 써보세요. VS Code와 인터페이스가 거의 같아서 적응이 빠르고, AI 기능은 매달 일정량 무료로 제공됩니다.

Cursor 설치 방법

  1. 1
    cursor.com 접속 → Download for [OS] 클릭

    Windows/Mac 모두 지원합니다.

  2. 2
    설치 후 실행 → GitHub 계정으로 로그인
  3. 3
    VS Code 설정 가져오기 옵션이 뜨면 Import (선택사항)

    기존 VS Code 설정을 그대로 쓸 수 있습니다.

  4. 4
    Node.js가 없다면 먼저 설치합니다

    nodejs.org → LTS 버전 다운로드 후 설치. 터미널에서 node -v로 확인합니다.

Step 2 — 프로젝트 생성

첫 프로젝트 만들기

Cursor 안의 터미널(Ctrl + `)에서 아래 명령어를 실행합니다. 무료 트랙과 동일합니다.

Terminal (Cursor 내장) 
# 원하는 위치로 이동
cd Desktop

# Vite + React 프로젝트 생성
npm create vite@latest my-app
# Framework: React / Variant: JavaScript 선택

cd my-app
npm install
npm run dev
# localhost:5173 에서 열립니다

프로젝트가 만들어지면, Cursor에서 해당 폴더를 엽니다. File → Open Folder → 방금 만든 폴더 선택.

이 다음부터가 무료 트랙과 달라집니다. Cursor에서 폴더를 열면, AI가 이 프로젝트의 모든 파일을 볼 수 있습니다. 이제 채팅창에서 대화하듯 수정을 요청하면 됩니다.

Step 3 — 핵심 원칙

코드보다 설계 먼저

유료 사용자가 가장 많이 하는 실수: 처음부터 "코드 짜줘"로 시작하는 것. 방향 없이 코드를 쌓다 보면 나중에 기능 추가가 어려워집니다.

대신 이렇게 시작해보세요. 먼저 설계를 받고, 그 다음에 코드를 요청합니다.

💬 1단계: 설계 먼저 요청하기
개발 동아리 신입부원을 위한 출석 체크 웹앱을 만들고 싶어.

기술 스택: React + Vite, 기본 CSS (Tailwind 없이)
기능:
- 이름 입력 → 출석 버튼 클릭 시 목록에 추가
- 중복 이름 방지
- 출석자 삭제 버튼
- localStorage로 새로고침 후에도 유지
배포: Vercel

지금 당장 코드는 필요 없고, 먼저:
1. 어떤 컴포넌트(파일)로 나눌지 구조를 알려줘
2. 각 파일이 하는 역할
3. 개발 순서 (무엇을 먼저 만들면 좋은지)
를 정리해줘.

왜 설계를 먼저 받나요? 설계를 먼저 잡으면 나중에 기능을 추가할 때 어디에 추가해야 할지 명확해지고, 오류가 났을 때 어느 파일을 봐야 하는지도 분명해집니다. AI에게도 "방향을 정해주면 더 좋은 코드가 나온다"는 느낌을 줄 수 있어요.

설계 후 코드 요청하기

💬 2단계: 설계 확인 후 코드 요청
위에서 잡은 구조대로 개발을 시작할게.
먼저 App.jsx와 AttendanceForm.jsx를 만들어줘.

이번 단계에서는 기능보다 UI 레이아웃만 먼저.
- 제목
- 이름 입력창
- 출석 버튼
- 출석자 목록 자리(빈 리스트)

스타일은 인라인으로 간단하게 넣어줘.
각 파일을 어디에 저장해야 하는지도 알려줘.
Step 4 — 개발 루프

4단계 루프 반복하기

유료 트랙에서도 이 루프를 반복합니다. 차이는 속도와 편의성뿐, 흐름은 동일합니다.

목표 정의

  • 이번 단계에서 만들 기능이 뭔지
  • 누가 쓰는 서비스인지
  • 이번 단계에서 하지 않을 것
  • 완성 기준 (어떻게 보이면 OK?)

구현 요청

  • 구조 / 역할 정의 요청
  • 첫 버전 코드 생성
  • 수정 범위 명확하게 지정
  • 이전 맥락 함께 제공

실행 & 검증

  • 브라우저에서 직접 실행
  • 기능 하나하나 테스트
  • 오류 발생 시 즉시 처리
  • 모바일에서도 확인

개선 요청

  • 디자인 / 색상 조정
  • 예외 처리 추가
  • localStorage 저장
  • 컴포넌트 분리 정리

Cursor에서 AI에게 요청하는 방법

Cursor에는 두 가지 방식이 있습니다.

💬

Chat 창 (Ctrl+L)

AI와 대화하며 코드를 제안받습니다. 제안을 수락하면 파일에 자동 반영됩니다. 여러 파일을 한 번에 수정할 수 있어요.

✏️

인라인 편집 (Ctrl+K)

파일을 열고 수정하고 싶은 부분을 선택한 후, Ctrl+K를 누르면 해당 부분만 AI가 수정해줍니다.

맥락을 잘 전달하기

Cursor는 프로젝트 파일을 볼 수 있지만, 현재 상황은 직접 알려주는 게 더 정확합니다.

❌ 맥락 없는 요청
중복 이름 방지 기능 추가해줘.
✅ 맥락 있는 요청
현재 React + Vite 프로젝트에서 출석 체크 기능까지 만들었어. App.jsx에 이름 입력 + 목록 표시가 있고, AttendanceForm.jsx에 폼이 있어. 이 구조를 유지하면서 중복 이름 방지 기능을 추가해줘.

역할을 부여하면 더 좋습니다

💬 역할 부여 요청 예시
너는 지금부터 프론트엔드 멘토야.
나는 React를 처음 배우는 대학생이고, 설명은 항상 입문자 눈높이로 해줘.

코드를 바꿀 때마다 왜 그렇게 바꾸는지 이유도 짧게 설명해줘.
파일을 수정하면 어느 파일을 바꿨는지 먼저 요약해줘.
실전 예제

출석 체크 앱 처음부터 끝까지

실제로 따라할 수 있는 순서입니다. 이 흐름을 한 번 경험하면 어떤 프로젝트든 응용할 수 있어요.

  1. 1
    설계 요청: 폴더 구조 + 컴포넌트 역할 + 개발 순서 받기
  2. 2
    App.jsx + AttendanceForm.jsx UI 레이아웃 먼저

    기능 없이 화면 구조만. 브라우저에서 레이아웃 확인.

  3. 3
    AttendanceList.jsx 출석자 목록 컴포넌트

    이름 목록을 출력하는 컴포넌트 추가.

  4. 4
    useState로 상태 연결 + 출석 추가 기능

    버튼 클릭 시 목록에 이름이 추가되는 기능 연결.

  5. 5
    중복 이름 방지 + 삭제 기능 추가
  6. 6
    localStorage 저장 추가

    새로고침 후에도 출석 목록이 남아있게.

  7. 7
    CSS 다듬기 + 모바일 반응형

    스마트폰에서도 잘 보이게 정리.

  8. 8
    빌드 테스트 + GitHub push + Vercel 배포

    "배포 전 체크리스트 뭐야?" AI에게 물어봐도 좋아요.

팁: 7단계가 끝나면 AI에게 "이 프로젝트를 README.md로 정리해줘"라고 요청해보세요. GitHub에 올릴 때 설명이 자동으로 생깁니다.

Step 5 — Git + 배포

변경사항 저장 & 배포

코드 작업이 어느 정도 되면 Git으로 저장하고 Vercel에 배포합니다. 유료 트랙도 이 과정은 동일합니다.

Git 기본 흐름

Terminal 
# 처음 한 번만 (저장소 연결)
git init
git add .
git commit -m "initial commit"
git branch -M main
git remote add origin GitHub 저장소 주소.git
git push -u origin main

# 이후 수정할 때마다
git add .
git commit -m "기능 추가: 중복 이름 방지"
git push
# → Vercel이 자동으로 재배포합니다

꼭 기억하세요: AI가 파일을 수정하기 전에 git add . && git commit을 해두세요. 그래야 AI가 이상하게 고쳐도 이전 상태로 되돌릴 수 있습니다.

Vercel 배포 (처음 한 번)

  1. 1
    vercel.com → GitHub 계정으로 로그인
  2. 2
    Add New → Project → GitHub 저장소 Import
  3. 3
    설정 기본값 그대로 → Deploy 클릭

    Vite 프로젝트는 자동 인식됩니다. 1~2분 후 배포 완료.

한 번에 너무 많이

🔎

이유를 묻지 않는 것

AI가 고쳐줬을 때 "왜 이렇게 했는지" 안 물어보면 기초가 약해집니다. 항상 이유를 요청하세요.

💾

Git 커밋을 안 하는 것

AI가 이상하게 고쳐도 되돌릴 수 있으려면, 자주 커밋해야 합니다.

핵심 기억: 유료 사용자 바이브코딩은 "프로젝트를 AI와 함께 공동제작하는 방식"에 가깝습니다. 생산성은 높아지지만, 방향 통제와 검토 책임은 항상 나에게 있습니다.
