Claude CLI, Codex CLI처럼 AI가 터미널에서 직접 내 프로젝트를 읽고 수정하는 환경. 처음엔 낯설지만, 익숙해지면 개발 속도가 완전히 달라집니다. 터미널 기초부터 배포까지 함께 알아봐요.

CLI란?

검은 창이 왜 강력한가

CLI는 Command Line Interface의 줄임말입니다. 쉽게 말하면, 마우스 없이 텍스트 명령어로 컴퓨터를 제어하는 방식입니다.

무섭게 보이지만, 사실 처음에 필요한 명령어는 6개가 전부입니다. 그리고 CLI 환경에서 AI와 함께 작업하면 이런 일이 가능해집니다.

🔍

프로젝트 전체 파악

AI가 내 폴더 구조를 직접 읽고 현재 상황을 스스로 파악합니다.

✏️

여러 파일 동시 수정

채팅창 복붙 없이 AI가 직접 파일을 만들거나 수정합니다.

🐛

실행 결과 기반 디버깅

오류 로그를 AI가 바로 보고 수정합니다. 훨씬 빠릅니다.

⚡

반복 작업 자동화

컴포넌트 분리, import 정리 같은 반복 작업을 빠르게 처리합니다.

세 가지 트랙 비교

F
무료 트랙 (복붙 방식) AI가 코드 생성 → 내가 복붙 → 브라우저 확인 → 오류 복붙 반복
P
유료 트랙 (AI 편집기) AI가 파일 직접 수정 가능 → 긴 대화 맥락 유지 → 설계부터 배포까지
C
CLI 트랙 (지금 여기) AI가 로컬 프로젝트를 직접 읽고 수정 → 가장 빠르고 강력한 방식

솔직하게 말하면: CLI는 강력하지만 진입 장벽이 있습니다. 터미널 명령어, 파일 경로, Git 상태 확인 같은 기초를 모르면 막힙니다. 이 페이지에서 그 기초부터 함께 배웁니다.

터미널 기초

이것만 알면 시작할 수 있어요

처음에 필요한 명령어는 딱 6개입니다. 외우려 하지 말고, 이 페이지를 참고하면서 쓰다 보면 자연스럽게 익혀집니다.

명령어
하는 일
예시
pwd
지금 내가 어느 폴더에 있는지 보여줌
/Users/me/Desktop/my-app
ls
현재 폴더 안에 뭐가 있는지 목록 보기
src/ public/ package.json
cd 폴더이름
해당 폴더로 이동
cd Desktop
cd ..
한 단계 위 폴더로 이동
cd .. (상위 폴더)
npm install
프로젝트에 필요한 패키지 설치
처음 프로젝트 받았을 때
npm run dev
개발 서버 실행 (브라우저에서 보임)
localhost:5173 에서 열기

팁: 터미널에서 막히면 AI에게 그대로 물어보면 됩니다. "지금 내 터미널 위치가 pwd 결과가 이래. 프로젝트 폴더로 어떻게 이동하면 돼?" 이렇게요.

터미널 여는 방법

🪟

Windows

시작 메뉴 → "Windows Terminal" 검색
또는 VS Code에서 Ctrl+`

🍎

Mac

Spotlight(Cmd+Space) → "Terminal" 검색
또는 VS Code에서 Ctrl+`

첫 프로젝트 만들기 (터미널에서)

Terminal 
# 1. 원하는 위치로 이동
cd Desktop   # 바탕화면으로 이동
pwd          # 위치 확인 (출력: /Users/me/Desktop)

# 2. Vite + React 프로젝트 생성
npm create vite@latest my-cli-app
# → Framework: React 선택
# → Variant: JavaScript 선택

# 3. 프로젝트 폴더로 이동 + 실행
cd my-cli-app
npm install
npm run dev
# 브라우저에서 http://localhost:5173 열기
Step 2 — CLI AI 도구 설치

Claude CLI 또는 Codex CLI 설치하기

CLI AI 도구는 터미널에서 직접 실행하는 AI입니다. 프로젝트 폴더 안에서 실행하면, AI가 그 폴더의 파일들을 읽고 수정할 수 있습니다.

Claude CLI 설치 (Claude Code)

  1. 1
    Node.js 18 이상 설치 확인

    node -v로 버전 확인. v18 미만이면 nodejs.org에서 최신 LTS 재설치.

  2. 2
    Claude CLI 설치

    아래 명령어를 터미널에서 실행합니다.

Terminal — Claude CLI 설치 
# Claude CLI 설치
npm install -g @anthropic-ai/claude-code

# 설치 확인
claude --version

# 처음 실행 (API 키 설정)
claude
# → 브라우저가 열리고 Anthropic 계정으로 로그인

비용 주의: Claude CLI는 Anthropic API를 사용하므로 사용량에 따라 비용이 발생할 수 있습니다. 처음엔 소량만 써보고, 사용 패턴을 파악한 후 본격적으로 사용하세요. Anthropic 대시보드에서 사용량을 확인할 수 있습니다.

Codex CLI 설치 (OpenAI)

Terminal — Codex CLI 설치 
# Codex CLI 설치
npm install -g @openai/codex

# OpenAI API 키 설정
# platform.openai.com → API Keys → Create new key
export OPENAI_API_KEY="sk-..."
# Windows는: set OPENAI_API_KEY=sk-...

# 실행 테스트
codex "hello"
Step 3 — 개발 워크플로우

AI CLI와 함께 개발하기

CLI AI 도구는 프로젝트 폴더 안에서 실행합니다. 그러면 AI가 그 폴더의 파일을 자동으로 읽고 맥락을 파악합니다.

Terminal — CLI AI 시작 
# 프로젝트 폴더로 이동
cd my-cli-app

# Claude CLI 실행
claude

# 또는 Codex CLI 실행
codex

# AI가 현재 폴더를 자동으로 읽기 시작합니다
# 이제 대화하듯 요청하면 됩니다

처음 요청할 때 중요한 것

CLI에서 AI에게 요청할 때는 작업이 끝나면 무엇이 달라졌는지 반드시 요약해달라고 해야 합니다. 그래야 어떤 파일이 바뀐 건지 입문자도 파악할 수 있습니다.

💬 CLI에서 이렇게 요청하기
이 React + Vite 프로젝트를 집중 타이머 앱으로 만들어줘.

기능:
- 25분 카운트다운 타이머
- 시작 / 일시정지 / 초기화 버튼
- 타이머 완료 시 "집중 완료!" 메시지

App.jsx를 수정하면 되고, 컴포넌트 파일도 새로 만들어도 괜찮아.
스타일은 깔끔하게, 모바일에서도 잘 보이면 좋겠어.

작업이 끝나면:
1. 어떤 파일을 만들거나 수정했는지
2. 각 파일이 무슨 역할인지
3. 다음에 내가 직접 수정하려면 어디를 보면 되는지
를 요약해줘.

전체 개발 흐름

01
Vite 프로젝트 생성 + 실행 확인 npm run dev로 기본 화면 확인
02
프로젝트 폴더에서 CLI AI 실행 claude 또는 codex 명령어로 AI 도구 열기
03
기능 설명 + 요약 요청 위 프롬프트처럼 구체적으로 + 작업 후 요약 요청
04
변경된 파일 직접 확인 AI가 요약해준 파일 목록을 VS Code에서 열어서 눈으로 확인
05
브라우저에서 실행 확인 npm run dev → 기능 하나하나 테스트
06
개선 요청 반복 디자인, 예외처리, localStorage 저장 등 추가
07
빌드 테스트 npm run build 통과 확인 후 배포 준비

오류가 났을 때

CLI에서는 오류 메시지를 AI가 바로 볼 수 있어서 훨씬 빠르게 해결됩니다.

Terminal — 빌드 오류 발생 시 
npm run build
# 오류가 나면 로그가 출력됩니다
# 그 로그를 그대로 AI에게 전달합니다
💬 빌드 오류 시 요청
npm run build를 실행했더니 오류가 났어.

오류 로그: [로그 전체 붙여넣기]

원인을 분석해줘. 그리고 최소한의 수정으로 해결해줘.
수정한 파일과 이유도 같이 알려줘.

핵심 습관: CLI가 편하다고 "알아서 다 고쳐줘"라고 하지 마세요. 입문자일수록 이유를 물어봐야 합니다. "왜 이렇게 수정하는지 설명해줘"를 항상 붙이세요.

Step 4 — Git으로 실수 보험

Git은 실수를 되돌리는 보험입니다

CLI 사용자에게 Git은 거의 필수입니다. AI가 파일을 수정해줬는데 결과가 이상할 때, Git이 없으면 이전 상태로 돌아갈 수 없습니다.

기본 Git 흐름

Terminal — Git 기본 사용 
# 현재 어떤 파일이 바뀌었는지 확인
git status

# 변경사항 기록 (저장)
git add .
git commit -m "feat: 타이머 기능 구현"

# 잘못됐을 때 되돌리기
git log               # 커밋 목록 보기
git checkout 커밋ID   # 그 시점으로 되돌아가기

꼭 기억하세요: AI에게 파일 수정을 요청하기 전에 반드시 git add . && git commit -m "수정 전 저장"을 해두세요. 그래야 AI가 이상하게 고쳐도 되돌릴 수 있습니다.

처음 Git 설정하기

Terminal — 최초 1회 설정 
# 이름과 이메일 설정 (GitHub 계정과 동일하게)
git config --global user.name "내이름"
git config --global user.email "내이메일@example.com"

# 프로젝트 폴더에서 Git 시작
git init
git add .
git commit -m "initial commit"

GitHub에 올리기

Terminal — GitHub 연결 
# GitHub 저장소 만든 후 연결
git remote add origin https://github.com/내계정/내저장소.git
git branch -M main
git push -u origin main

# 이후 수정할 때마다
git add .
git commit -m "변경 내용"
git push
# → Vercel이 자동으로 재배포합니다
Step 5 — 배포

GitHub → Vercel 배포

배포 방법은 다른 트랙과 동일합니다. 단, 배포 전에 반드시 빌드 테스트를 먼저 합니다.

Terminal — 배포 전 체크 
# 1. 빌드 먼저 확인 (이게 통과해야 Vercel도 성공)
npm run build
# 성공 시 dist 폴더가 생성됨

# 2. GitHub에 올리기
git add .
git commit -m "complete: 타이머 앱 완성"
git push
# → Vercel 자동 재배포

개발 서버 vs 빌드: npm run dev로는 잘 되는데 npm run build에서 실패하는 경우가 있습니다. 반드시 배포 전에 빌드를 먼저 테스트하세요.

배포 후 수정 사이클

1
코드 수정 CLI AI와 함께 또는 직접 수정
2
git commit + push 변경사항 기록 후 GitHub 업로드
3
Vercel 자동 재배포 1~2분 기다리면 새 버전 자동 배포 완료
핵심 마인드셋

CLI 사용자에게 가장 중요한 세 가지

CLI가 강력한 만큼, 검토하지 않고 넘어가는 위험도 커집니다. 이 세 가지를 기억하세요.

01

맡기되 검토하기

AI가 만든 결과물을 무조건 믿지 마세요. 항상 직접 실행하고 눈으로 확인합니다.

02

이해를 우선으로

잘 돌아가는 것보다, 왜 그렇게 돌아가는지 아는 게 더 중요합니다. 항상 이유를 물어보세요.

03

작게 자주 확인

큰 작업을 한 번에 맡기지 마세요. 기능 하나 → 확인 → 커밋 → 다음 기능 순서로.

이것만은 하지 마세요: AI가 파일을 알아서 수정해줬는데 실행도 안 해보고 바로 push하는 것. 반드시 npm run dev로 먼저 확인하고, npm run build도 통과시킨 다음에 push하세요.

자주 하는 실수와 해결법

  1. !
    AI가 너무 많은 파일을 바꿔서 뭐가 뭔지 모르겠어요

    다음 요청부터 범위를 명시하세요. "App.jsx만 건드려줘" 또는 "src/components 폴더만 수정해줘"처럼.

  2. !
    코드가 잘 돌아가는데 이해가 전혀 안 돼요

    이건 매우 흔합니다. AI에게 "입문자 눈높이로 각 파일 역할을 설명해줘"라고 요청하세요.

  3. !
    터미널에서 명령어가 안 먹혀요

    pwd로 현재 위치를 먼저 확인하세요. 프로젝트 폴더 안에 있어야 npm 명령어가 동작합니다.

추천 첫 프로젝트 (CLI 트랙)

  • 집중 타이머 앱 (25분 뽀모도로)
  • 동아리 출석 체크 앱
  • 간단한 메모 앱 (localStorage 저장)
  • 날씨 앱 (Open Weather API 활용)
  • 개인 포트폴리오 페이지

마지막으로: CLI 바이브코딩은 처음엔 어려워 보여도, 한 번 익숙해지면 개발 속도와 이해도가 동시에 올라갑니다. "검은 창"이 친구가 되는 순간, 개발이 완전히 달라집니다.
